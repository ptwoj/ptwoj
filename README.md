## 🚀 Projects

### 🐾 [PetClinic AWS Migration Project](https://github.com/hyh528/PetClinic-AWS-Migration)

<p>
  <img src="https://img.shields.io/badge/Spring_Boot-6DB33F?style=flat-square&logo=springboot&logoColor=white" alt="Spring Boot"/>
  <img src="https://img.shields.io/badge/AWS-FF9900?style=flat-square&logo=amazonaws&logoColor=white" alt="AWS"/>
  <img src="https://img.shields.io/badge/Terraform-7B42BC?style=flat-square&logo=terraform&logoColor=white" alt="Terraform"/>
  <img src="https://img.shields.io/badge/Docker-2496ED?style=flat-square&logo=docker&logoColor=white" alt="Docker"/>
  <img src="https://img.shields.io/badge/Aurora-527FFF?style=flat-square&logo=amazonrds&logoColor=white" alt="Aurora"/>
</p>

> **Spring PetClinic Microservices on-premise to AWS Cloud Migration**
>
> **Role:** Data & Migration Lead (데이터베이스 및 마이그레이션 총괄)

온프레미스 MySQL DB를 AWS Aurora로 이전하고, MSA 설정 아키텍처를 클라우드 네이티브 방식으로 전환하는 프로젝트.  
데이터베이스 설계, IaC 구축, 마이그레이션, 안정화까지 데이터 관련 모든 기술 책임을 담당.

#### 📊 주요 성과

| 성과 지표 | Before | After | 개선율 |
|---------|--------|-------|-------|
| **오류율** | 90% | 0% | **90% ↓** |
| **배포 시간** | 수동 1시간+ | Terraform 20분 | **100% ↓** |
| **인프라 재현 시간** | N/A | 20분 | **신규 구축** |

#### ✨ 핵심 기술 성과

<details>
<summary><b>1️⃣ 운영 수준의 Aurora DB 인프라 설계 (Terraform 100%)</b></summary>

<br>

- **Infrastructure as Code 구현**
  - Terraform으로 Aurora 클러스터(MySQL 8.0) 완전 자동화 구축
  - `terraform_remote_state`를 통해 Network/Security 팀 인프라와 안전하게 연동
  - DB 파라미터(`utf8mb4`, `Asia/Seoul`, `max_connections=500`) 코드 관리
  - `slow_query_log` 활성화로 성능 분석 기반 마련

- **운영 효율성 극대화**
  - Public Subnet의 Bastion Host(EC2)를 코드로 구축
  - DB 연결 검증 및 문제 해결 효율성 향상
  - 어떤 환경에서든 **20분 내 전체 인프라 재현 가능**

</details>

<details>
<summary><b>2️⃣ 안전하고 확장 가능한 설정 아키텍처 구현</b></summary>

<br>

- **중앙화된 설정 관리 (AWS Parameter Store)**
  - 공통/서비스별 설정을 계층적으로 분리 (`petclinic/common/`, `petclinic/dev/{service}/`)
  - 설정 변경 시 **재배포 없이 즉시 반영** 가능한 유연한 구조

- **보안 강화 (AWS Secrets Manager + KMS)**
  - DB 비밀번호 등 민감 정보를 Secrets Manager로 분리
  - Aurora 생성 시 `manage_master_user_password = true` 옵션으로 **코드 노출 없이 자동 생성/교체**
  - AWS KMS 키를 별도 생성하여 비밀번호 암호화, 보안 강화

</details>

<details>
<summary><b>3️⃣ 데이터 마이그레이션 성공 및 무결성 검증</b></summary>

<br>

- **실전 문제 해결**
  - ❌ **인코딩 오류(UTF-16)** → ✅ UTF-8 변환 자동화
  - ❌ **스키마 불일치(Unsigned Mismatch)** → ✅ 스키마 수정 및 검증
  - ❌ **트랜잭션 롤백(Ghost Data)** → ✅ 안정적인 임포트 절차 수립

- **데이터 무결성 100% 보장**
  - 테이블별 Row Count 비교 및 CRUD 쿼리 테스트
  - 마이그레이션 절차 문서화 (`docker cp` → `vi` → `SOURCE`)

</details>

<details>
<summary><b>4️⃣ 보안 아키텍처 구축</b></summary>

<br>

- **네트워크 보안**
  - VPC, Subnet, NACL, Security Group을 활용한 3-Tier 망 분리
  - Security Group으로 각 레이어 간 트래픽 엄격 통제

- **멀티 레이어 보안 체계**
  - **AWS WAF**: 웹 공격 방어
  - **Honeypot**: 공격자 패턴 수집 및 분석
  - **ELK Stack**: 비정상 접근 로그 실시간 분석 및 시각화

</details>

<details>
<summary><b>5️⃣ 모니터링 & 성능 최적화</b></summary>

<br>

- **CloudWatch 기반 모니터링**
  - DB 핵심 지표 대시보드 구축 (`CPUUtilization`, `DatabaseConnections`, `SelectLatency`)
  - 임계치 기반 경보(Alarm) 설정으로 장애 예방 체계 완성

- **성능 최적화**
  - `slow_query_log` 분석 기반 쿼리 튜닝
  - 인덱스 전략 수립 및 적용

</details>
